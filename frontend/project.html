Miten Docker-projektimme toimii

Tämä sivu selittää lyhyesti miten koko projektimme toimii Dockerin sisällä.

1. Frontend

Frontend on yksinkertainen staattinen Nginx-palvelin, joka näyttää HTML-sivuja. Dockerfile kopioi vain HTML-tiedoston ja avaa portin 80. Jiska tulee lisäämään omia hommian ja lisätään sitten tälle sivulla tietoa.

2. Notes-API

Backend-palvelu on tehty FastAPI:lla. Palvelu pyörii Uvicorn-palvelimella ja kuuntelee porttia 8000.

3. Proxy (Traefik)

Traefik toimii käänteisenä proxyna. Se ohjaa liikennettä eri konttien välillä ja voi hallita myöhemmin myös SSL/TLS-sertifikaatteja. Ei tällä hetkellä mukana projektissa vielä, koska windows ei halua tehdä yhteistyötä D:

4. Docker Compose

Kaikki palvelut käynnistetään yhdellä komennolla: 

docker compose up --build (tarkista, että docker desktop on päällä!)
